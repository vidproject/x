Tips

  • The viewer starts with the full searchable catalog. It loads full tweet records as rows come into view, are opened, or are reached from a shared link. Use the lightning button only when you want every full Parquet row downloaded for fast browsing.
  • Search defaults to all fields. Change the field selector beside it to search only text, account, tags, mentions, media descriptions, or another single field.
  • ? in search matches any single character; * matches any run.
  • Exclude tags right in the search box with -tag:. Use -tag:namespace:slug to drop one exact tag, or -tag:namespace to drop a whole namespace. Add more -tag: tokens to exclude additional tags; the rest of the box still runs as a normal search. Example: deportation -tag:genre:music-video -tag:format.
  • Click the filter button on any column header to filter by that column's unique values.
  • Click any row to open the full tweet detail panel on the right.
  • Use Columns to toggle visibility, including Deletion detected.
  • Export / Share offers six actions. Share copies a link; Export downloads a CSV. Each comes in three scopes: Selection (only the rows you tick in the left checkbox column), Page (just the current page), or Query (every row matching the active filters and search). Share · Selection hands someone a link to exactly the items you picked — tick across several searches to gather a subset, then share it.
  • The URL also updates as you filter; bookmark or share it to come back to the same view.
  • This is a public archive. Captures mirror what X served the browser at capture time.
